## How to Use This Matrix

| Column | Description |
|--------|-------------|
| Jurisdiction | Country or state |
| Primary Law | Main applicable privacy legislation |
| Year | Effective year of current law |
| Consent Model | Opt-in (must get consent) vs Opt-out (assumed consent unless refused) |
| Right to Delete | Whether individuals have a legal right to deletion/erasure |
| Breach Timeline | Maximum days to notify regulators after discovering breach |
| Data Residency | Whether health data must remain in-country |
| Cross-Border Rules | Conditions for international data transfers |
| DPO/Officer Required | Whether a dedicated privacy officer is mandatory |
| Max Penalties | Maximum financial penalties |
| Healthcare Notes | Health-data-specific provisions |

## Americas

### United States — Federal

| Element | Federal (HIPAA) |
|---------|----------------|
| Primary Law | HIPAA (1996) + HITECH (2009) |
| Year | 2009 (HITECH amendments) |
| Consent Model | **Opt-out** for TPO; Opt-in (authorization) for other uses |
| Right to Delete | No (right to restrict, not delete) |
| Breach Timeline | 60 days to HHS; 60 days to individuals |
| Data Residency | None at federal level |
| Cross-Border Rules | BAA required; no explicit territorial restriction |
| DPO/Officer Required | Privacy Officer required (not DPO per se) |
| Max Penalties | Up to $1.9M per violation category per year (HITECH) |
| Healthcare Notes | PHI definition; 42 CFR Part 2 for SUD records |

### United States — California

| Element | California |
|---------|-----------|
| Primary Law | CCPA (2018) + CPRA (2020) + CMIA (1981, amended) |
| Year | CPRA effective 2023 |
| Consent Model | Opt-out (CCPA/CPRA); Opt-in for sensitive data under CPRA |
| Right to Delete | Yes (CPRA Art. 1798.105) |
| Breach Timeline | Most expedient time; no specific days — notify AG if 500+ Californians |
| Data Residency | None |
| Cross-Border Rules | Service provider contracts required |
| DPO/Officer Required | No (but recommended) |
| Max Penalties | $2,500 per unintentional violation; $7,500 intentional |
| Healthcare Notes | CMIA applies to medical information; stricter than CCPA; HIPAA-exempt data still covered by CMIA |

### Canada — Ontario (PHIPA)

| Element | Ontario |
|---------|--------|
| Primary Law | Personal Health Information Protection Act (PHIPA, 2004, amended 2020) |
| Year | 2004 |
| Consent Model | Opt-in (express or implied consent for healthcare purposes) |
| Right to Delete | Limited (correction, not deletion) |
| Breach Timeline | Notify IPC; notify individuals at risk of harm |

| Cross-Border Rules | Contractual requirements + equivalent protection |

| Healthcare Notes | Applies to "health information custodians"; broader than HIPAA; electronic records mandatory |

### Australia

| Element | Australia |
|---------|----------|
| Primary Law | Privacy Act 1988 + Australian Privacy Principles + My Health Records Act 2012 |
| Year | 1988 (APP revised 2014) |
| Consent Model | Opt-in for sensitive data (health is sensitive) |

| Healthcare Notes | My Health Records Act creates additional obligations for MHR participants; Australian Digital Health Agency oversight |

## Europe

### European Union

| Element | EU |
|---------|---|
| Primary Law | GDPR (2016/679) + EHDS Regulation 2025/327 |
| Year | 2018 (GDPR effective) |
| Consent Model | Opt-in (explicit for health data) |
| Right to Delete | Yes (Art. 17) with healthcare exemptions |
| Breach Timeline | **72 hours** to supervisory authority |
| Data Residency | No general requirement; EHDS promotes data within EU |
| Cross-Border Rules | Adequacy, SCCs, BCRs, or derogations |
| DPO/Officer Required | **Yes** for large-scale health data processing |
| Max Penalties | €20M or 4% global annual turnover (whichever higher) |
| Healthcare Notes | Art. 9(2)(h) for healthcare; EHDS for cross-border; see dedicated GDPR and EHDS guides |

## Key Implementation Notes

### Our API Configurable Controls Per Jurisdiction

| Control | Configuration Point | Affected Jurisdictions |
|---------|--------------------|-----------------------|
| Default consent model | `Consent.policyRule` + `Organization.extension.defaultConsentModel` | All |
| Data residency enforcement | Infrastructure deployment region + `Organization.extension.dataResidency` | Alberta (CA), KSA |
| Breach notification timer | Compliance monitoring module alert thresholds | All — 72h for EU/UK/BR; 60d for US/HIPAA |
| DPO contact | `Organization.contact.purpose = "PATINF"` | EU, UK, PH, KR, TH, ZA, NG |
| Deletion support | `Patient.$purge` operation availability flag | All except US-HIPAA |
| Retention enforcement | `RetentionPolicy` resource per jurisdiction | All |

### Data Residency Infrastructure Notes

| Requirement | Jurisdictions | Implementation |
|-------------|--------------|----------------|
| Data cannot leave country | KSA, Alberta (CA) | Deploy dedicated regional instance |
| Data cannot leave region | EU (EHDS preference) | EU-West or EU-Central deployment |
| Data transfer requires contract | Most jurisdictions | Standard data processing agreement template |
| No restriction | US-Federal, Australia (private sector), Colombia | Any compliant cloud region |
