## W3C PROV-DM Core Concepts

### Three Core Types

| PROV Type | Symbol | Definition | Healthcare Example |
|-----------|--------|-----------|-------------------|
| **Entity** | `e:` | A physical, digital, conceptual, or other kind of thing with some fixed aspect | A clinical observation record, a medication order, a patient document |
| **Activity** | `a:` | Something that occurs over a period of time and acts upon or with entities | Writing a note, signing an order, running a lab test |
| **Agent** | `ag:` | Something that bears some form of responsibility for an activity | A practitioner, an organization, a software system |

### Core PROV Relations

| Relation | Type | Connects | Meaning |
|----------|------|---------|---------|
| `wasGeneratedBy` | Entity ← Activity | Entity was created by activity | Lab result generated by lab test activity |
| `used` | Activity → Entity | Activity used entity as input | Order-sign activity used medication draft |
| `wasDerivedFrom` | Entity ← Entity | Entity derived from another entity | Discharge summary derived from encounter notes |
| `wasAttributedTo` | Entity ← Agent | Agent is responsible for entity | Observation attributed to the ordering physician |
| `wasAssociatedWith` | Activity ← Agent | Agent associated with activity | Signing activity associated with Dr. Smith |
| `actedOnBehalfOf` | Agent ← Agent | Agent acted on behalf of another | Nurse acted on behalf of physician |
| `wasInformedBy` | Activity ← Activity | Activity was informed by another activity | Discharge planning informed by admission assessment |
| `wasStartedBy` | Activity ← Entity | Entity started the activity | Encounter note started by encounter |
| `wasEndedBy` | Activity ← Entity | Entity ended the activity | Prescription ended by dispensing event |
| `wasInvalidatedBy` | Entity ← Activity | Activity invalidated entity | Amendment invalidated previous note version |
| `specializationOf` | Entity → Entity | Entity is specialization of a more general entity | Specific observation is specialization of observation class |
| `alternateOf` | Entity ↔ Entity | Entities represent same thing | Two records representing the same patient |

## Our Provenance Model → W3C PROV Mapping

### ProvenanceRecord → prov:Activity

| Our Field | W3C PROV | Description |
|-----------|---------|-------------|
| `ProvenanceRecord.id` | `prov:identifier` | Unique ID of the provenance activity |
| `ProvenanceRecord.type` | `prov:Activity.type` | Type of activity (create, update, delete, amend, transmit, etc.) |
| `ProvenanceRecord.recorded` | `prov:startTime` + `prov:endTime` | When the activity occurred |
| `ProvenanceRecord.activity` | `prov:Activity` class | The clinical activity being recorded |
| `ProvenanceRecord.reason` | `prov:Activity.label` | Reason for the activity |
| `ProvenanceRecord.policy` | `prov:qualifiedAssociation.hadPlan` | Policy or protocol followed |
| `ProvenanceRecord.location` | Activity location attribute | Where the activity occurred |

### ProvenanceAgent → prov:Agent

| Our Field | W3C PROV | Description |
|-----------|---------|-------------|
| `ProvenanceAgent.who` | `prov:Agent.identifier` | Reference to Practitioner, Organization, Device |
| `ProvenanceAgent.onBehalfOf` | `prov:actedOnBehalfOf` | Delegating agent |
| `ProvenanceAgent.type` | `prov:Agent.type` | author, custodian, enterer, transmitter, verifier, informant, assembler |
| `ProvenanceAgent.role` | `prov:qualifiedAssociation.hadRole` | Clinical role of the agent |
| `ProvenanceAgent.requestor` | `prov:Agent` (requestor role) | Who triggered the activity |

### ProvenanceEntity → prov:Entity

| Our Field | W3C PROV | Description |
|-----------|---------|-------------|
| `ProvenanceEntity.role` | `prov:Entity.type` | derivation, revision, quotation, source |
| `ProvenanceEntity.what` | `prov:Entity.identifier` | Reference to FHIR resource |
| `ProvenanceEntity.agent` | `prov:wasAttributedTo` | Agent responsible for the entity |

### Activity Type Vocabulary

| Our Activity Code | W3C PROV Concept | FHIR v3 Code | Description |
|------------------|-----------------|-------------|-------------|
| `CREATE` | `prov:Generation` | `CREATE` | Initial resource creation |
| `UPDATE` | `prov:Revision` | `UPDATE` | Modification of existing resource |
| `DELETE` | `prov:Invalidation` | `DELETE` | Deletion or inactivation |
| `AMEND` | `prov:Revision` | `AMEND` | Clinical amendment |
| `TRANSMIT` | `prov:Derivation` | `TRANS` | Transmission to external system |
| `VERIFY` | `prov:Association` | `VRFCRSP` | Verification/attestation |
| `APPEND` | `prov:Derivation` | `APND` | Appending to a record |
| `ACCESS` | `prov:Use` | `REMOTE` | Read access |
| `EXPORT` | `prov:Derivation` | `TRANS` | Data export |
| `IMPORT` | `prov:Derivation` | `INST` | Data import |

## PROV Relations Mapping

### wasGeneratedBy (Entity ← Activity)

**Scenario:** A new Observation is created from a lab test activity.

```
FHIR Model:
  Observation (entity) + Provenance.activity = "create" + Provenance.agent = lab system

PROV-N:
  wasGeneratedBy(obs-123, lab-test-activity, 2026-04-14T14:30:00Z)
  wasAssociatedWith(lab-test-activity, lab-system-agent, -)
```

**FHIR Provenance fields used:**
- `Provenance.target` = the generated entity (Observation)
- `Provenance.activity.code` = create
- `Provenance.agent` = the generating system

### wasDerivedFrom (Entity ← Entity)

**Scenario:** A Discharge Summary is derived from the Encounter and all its notes.

```
FHIR Model:
  Composition.relatesTo[type=derived-from] → source encounters/documents

PROV-N:
  wasDerivedFrom(discharge-summary, encounter-note-1, -, -, -)
  wasDerivedFrom(discharge-summary, encounter-note-2, -, -, -)
  wasDerivedFrom(discharge-summary, lab-results-bundle, -, -, -)
```

**FHIR Provenance fields used:**
- `Provenance.entity[role=source].what` = source resources
- `Provenance.target` = derived document

### wasAttributedTo (Entity ← Agent)

**Scenario:** A clinical note is attributed to the author.

```
FHIR Model:
  Composition.author[] = [Practitioner/dr-123]
  Provenance.agent[type=author].who = Practitioner/dr-123

PROV-N:
  wasAttributedTo(composition-456, dr-123)
```

### wasAssociatedWith (Activity ← Agent)

**Scenario:** A medication order signing is associated with the prescribing physician.

```
FHIR Model:
  Provenance.activity = order-sign
  Provenance.agent[type=author].who = Practitioner/dr-456

PROV-N:
  wasAssociatedWith(order-sign-activity, dr-456, -)
```

### actedOnBehalfOf (Agent ← Agent)

**Scenario:** A nurse acts on behalf of a physician when entering a telephone order.

```
FHIR Model:
  Provenance.agent[0] = { type: enterer, who: Practitioner/nurse-789 }
  Provenance.agent[0].onBehalfOf = Practitioner/dr-456

PROV-N:
  actedOnBehalfOf(nurse-789, dr-456, telephone-order-activity)
```

### used (Activity → Entity)

**Scenario:** The order-sign activity used the draft MedicationRequest.

```
FHIR Model:
  Provenance.entity[role=source].what = MedicationRequest/draft-rx-123

PROV-N:
  used(order-sign-activity, draft-rx-123, 2026-04-14T12:00:00Z)
```

## FHIR Provenance Alignment

### FHIR Provenance Resource is Based on W3C PROV

The FHIR Provenance resource was designed to align with W3C PROV:

| FHIR Provenance Element | W3C PROV Equivalent |
|------------------------|---------------------|
| `Provenance` (resource) | `prov:Activity` |
| `Provenance.target` | Entity generated by the activity (`wasGeneratedBy`) |
| `Provenance.occurred[x]` | `prov:startTime` / `prov:endTime` |
| `Provenance.recorded` | When the provenance record itself was created |
| `Provenance.activity` | `prov:Activity.type` |
| `Provenance.agent` | `prov:Agent` associated via `wasAssociatedWith` |
| `Provenance.agent.onBehalfOf` | `prov:actedOnBehalfOf` |
| `Provenance.entity` | `prov:Entity` used by the activity |
| `Provenance.entity.role = source` | `prov:used` |
| `Provenance.entity.role = revision` | `wasDerivedFrom` (revision) |
| `Provenance.entity.role = quotation` | `wasDerivedFrom` (quotation) |
| `Provenance.entity.role = derivation` | `wasDerivedFrom` (general) |
| `Provenance.entity.role = removal` | `prov:Invalidation` |
| `Provenance.signature` | Digital attestation of the provenance record |
| `Provenance.reason` | `prov:Activity.label` or purpose attribute |
| `Provenance.location` | Activity location attribute |

## Serialization Formats

### PROV-N (Human-Readable Notation)

```
document
  prefix ex <https://api.monobase.health/fhir/>
  prefix prov <http://www.w3.org/ns/prov#>

  entity(ex:MedicationRequest/rx-789)
  entity(ex:Encounter/encounter-456)
  activity(ex:Provenance/prov-001, 2026-04-14T12:00:00Z, 2026-04-14T12:00:00Z)
  agent(ex:Practitioner/dr-smith)
  agent(ex:Organization/general-hospital)
  agent(ex:Device/monobase-ehr-system)

  wasGeneratedBy(ex:MedicationRequest/rx-789, ex:Provenance/prov-001, 2026-04-14T12:00:00Z)
  wasAssociatedWith(ex:Provenance/prov-001, ex:Practitioner/dr-smith, -)
  actedOnBehalfOf(ex:Practitioner/dr-smith, ex:Organization/general-hospital, -)
  wasDerivedFrom(ex:MedicationRequest/rx-789, ex:Encounter/encounter-456, ex:Provenance/prov-001, -, -)

endDocument
```

```json

      "prov:startTime": "2026-04-14T12:00:00Z",
      "prov:endTime": "2026-04-14T12:00:00Z",
      "prov:type": "prov:Create"
    }
  },
  "agent": {
    "ex:Practitioner/dr-smith": {},
    "ex:Organization/general-hospital": {}
  },
  "wasGeneratedBy": {
    "_:gen1": {
      "prov:entity": "ex:MedicationRequest/rx-789",
      "prov:activity": "ex:Provenance/prov-001",
      "prov:time": "2026-04-14T12:00:00Z"
    }
  },
  "wasAssociatedWith": {
    "_:assoc1": {
      "prov:activity": "ex:Provenance/prov-001",
      "prov:agent": "ex:Practitioner/dr-smith"
    }
  },
  "actedOnBehalfOf": {
    "_:behalf1": {
      "prov:delegate": "ex:Practitioner/dr-smith",
      "prov:responsible": "ex:Organization/general-hospital"
    }
  }

```

### PROV-O (RDF/OWL)

```turtle
@prefix prov: <http://www.w3.org/ns/prov#> .
@prefix ex: <https://api.monobase.health/fhir/> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .

ex:MedicationRequest/rx-789 a prov:Entity ;
    prov:wasGeneratedBy ex:Provenance/prov-001 ;
    prov:wasAttributedTo ex:Practitioner/dr-smith ;
    prov:wasDerivedFrom ex:Encounter/encounter-456 .

ex:Provenance/prov-001 a prov:Activity ;
    prov:startedAtTime "2026-04-14T12:00:00Z"^^xsd:dateTime ;
    prov:endedAtTime "2026-04-14T12:00:00Z"^^xsd:dateTime ;
    prov:wasAssociatedWith ex:Practitioner/dr-smith .

ex:Practitioner/dr-smith a prov:Agent ;
    prov:actedOnBehalfOf ex:Organization/general-hospital .
```

### Export Endpoint

```http
GET /fhir/Provenance/{id}/$prov

Accept: application/json               # PROV-JSON
Accept: text/turtle                    # PROV-O (Turtle)
Accept: application/ld+json            # PROV-O (JSON-LD)
```

## Healthcare Provenance Patterns

### Pattern 1: Clinical Documentation Chain

```
Patient consultation (Encounter)
    ↓ wasGeneratedBy
Progress note (Composition) [wasAttributedTo dr-smith]
    ↓ wasDerivedFrom
Discharge summary (Composition) [wasAttributedTo dr-jones]
    ↓ wasGeneratedBy
Transmitted document (DocumentReference) [transmitted to referring provider]
```

### Pattern 2: Lab Result Provenance

```
Lab order (ServiceRequest) [wasAttributedTo dr-smith]
    ↓ used by
Lab test activity (AuditEvent/Provenance)
    ↓ wasGeneratedBy
Lab result (Observation) [wasAttributedTo lab-system]
    ↓ wasGeneratedBy (when finalized by pathologist)
DiagnosticReport [wasAttributedTo pathologist-jones]
```

### Pattern 3: Amendment Chain

```
Original note (Composition v1) [wasAttributedTo dr-smith]
    ↓ wasRevisionOf (wasDerivedFrom, role=revision)
Amended note (Composition v2) [wasAttributedTo dr-smith, amendment-reason="Incorrect medication dose"]
```

### Pattern 4: Consent-Driven Disclosure

```
Consent (active, explicit) [wasAttributedTo patient]
    ↓ qualifiedAssociation.hadPlan
Disclosure activity (AuditEvent) [wasAssociatedWith external-provider, purpose=TREAT]
    ↓ wasGeneratedBy
Transmitted records (Bundle) [wasAttributedTo monobase-system]
```
