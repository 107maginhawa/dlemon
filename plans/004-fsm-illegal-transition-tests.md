## Why this matters

The FSM coverage engine (`scripts/coverage/fsm-matrix.ts`) reports **23 illegal state transitions that no test asserts are rejected** — e.g. `Treatment: declined → performed`, `Visit: locked → active`, `CephLandmark: locked → not_placed`. These are exactly the dangerous edges: a regression that accidentally made one of them *legal* (e.g. letting a declined treatment be marked performed, or a locked clinical visit reopened) would ship silently because nothing proves the guard rejects it. Each transition map (`*_TRANSITIONS`) is the single source of truth the handlers consult, so a fast, deterministic, DB-free property test asserting these edges are absent from the map closes the gap and arms the `Coverage Ratchet` CI gate against future drift.

### Step 1: Treatment — add explicit illegal-edge assertions

In `treatment.fsm.property.test.ts`, add a new `describe` block (after the existing ones) that asserts each Treatment illegal-uncovered edge is rejected by the map. Use the existing `isValidTransition` helper already defined in that file:

```ts
describe('illegal Treatment transitions are rejected (FSM coverage)', () => {
  const ILLEGAL: Array<[DentalTreatmentStatus, DentalTreatmentStatus]> = [
    ['declined', 'performed'],
    ['declined', 'verified'],
    ['diagnosed', 'verified'],
    ['dismissed', 'performed'],
    ['performed', 'declined'],
    ['verified', 'declined'],
    ['verified', 'diagnosed'],
  ];
  for (const [from, to] of ILLEGAL) {
    test(`${from} -> ${to} is not a legal transition`, () => {
      expect(isValidTransition(from, to)).toBe(false);
    });
  }
});
```
