# Imaging + Cephalometric Analysis — Standards & Experience Review
> Review date 2026-06-02 · Depth: DEEP

## 1. What we have   (cite spec + code; be specific about which analyses/landmarks/measurements exist)

**Two cephalometric analyses are implemented**, dispatched by `computeAnalysis(analysisType, …)` in `packages/ceph-math/src/index.ts` (`ANALYSIS_TYPES = ['steiner_hybrid_sn', 'ricketts']`):

- **Steiner-hybrid-SN** (`computeCephAnalysis`, `packages/ceph-math/src/index.ts`) — Sella–Nasion-referenced. Emits 17 metrics: `sna, snb, anb, convexity_napog, sn_gome, facial_angle_sn, y_axis_sn, u1_sn, impa, u1_na_angle, l1_nb_angle, interincisal, u1_na_mm, l1_nb_mm, overjet, overbite, axis_tilt_deg`. ANB is signed (`sna − snb`), never abs — Class III safe (D-M). Linear metrics gated on calibration (`mmMetric` returns null when `spacingMm` falsy).
- **Ricketts** (`packages/ceph-math/src/ricketts.ts`) — genuinely distinct Frankfort-Horizontal frame (Po-Or) + A-Pog facial line. Emits 6: `facial_angle, mandibular_plane_fh, convexity_mm, l1_apog_angle, l1_apog_mm, interincisal`. Metrics needing Basion/Pterygoid (facial axis) intentionally omitted, not fabricated.

**16 landmarks** (`LANDMARK_CODES`, single source of truth): S, N, A, B, ANS, PNS, Go, Po, Me, Or, Pog, Gn, U1T, U1A, L1T, L1A. Gn is *constructed* from Pog+Me midpoint when absent (D-P). Coordinates are stored image-space pixels (`imaging_ceph_landmark.x/y` real, D-C), not screen-space.

**Normative values are stored and displayed with mean ± SD and signed z-deviation** (`packages/ceph-math/src/norms.ts`). Norms are *keyed by analysis type* (`CEPH_NORMS.steiner_hybrid_sn` vs `.ricketts`) so switching analyses swaps the reference set — explicitly to prevent cross-analysis norm bleed (patient-safety). `classifyDeviation()` → normal ≤1 SD / mild 1–2 SD / severe >2 SD, with signed delta always reported (UI never relies on color alone). `CephMeasurementsPanel.tsx` renders amber (1–2 SD) / red (>2 SD) chips with `title="Norm {mean}±{sd} ({source})"`; deliberately **no green for "normal"**.

**Backend ceph handlers** (`services/api-ts/src/handlers/dental-imaging/`): `createCephReport`, `getCephReport`, `getCephAnalysis`, `recomputeCephAnalysis`, `updateCephLandmark`, `batchUpsertCephLandmarks`, `listCephLandmarks`, `deleteCephLandmark`. Three tables (`imaging_ceph.schema.ts`): `imaging_ceph_landmark` (FSM `not_placed→placed→confirmed→locked`, no back-edges), `imaging_ceph_analysis` (measurements JSONB + D-J calibration provenance: value/method/at/by), `imaging_ceph_report` (immutable append-only versioned snapshot, D-I). Report generation gated on A/B/Go/Po confirmed (`CEPH_REPORT_GATE_LANDMARKS`, D-L). All ceph features are **addon-tier gated** (`recomputeCephAnalysis` line 55; study-create blocks cephalometric modality at free/basic, `createImagingStudy` line 59).

**Recompute preconditions are enforced** (`recomputeCephAnalysis.ts`): missing reference landmarks → 422 `INSUFFICIENT_LANDMARKS` (S,N for Steiner; Po,Or,N for Ricketts); uncalibrated → 422 `NOT_CALIBRATED`. Does not silently return all-null.

**Generic imaging**: `imaging_study` / `imaging_study_image` / `imaging_study_tooth` (join) / `imaging_annotation` (`imaging.schema.ts`). Modality enum: periapical, bitewing, panoramic, cephalometric, **cbct**, intraoral_photo, extraoral_photo, other. Findings (`imaging_finding.schema.ts`): 15 types (caries, bone_loss, periapical_lesion, …), FSM `draft→confirmed→resolved`, linked to `toothNumber` (1–48) + `surfaces[]` + optional `annotationId`/`treatmentId`/`visitId`. Calibration (`updateImageCalibration`) is available at all tiers; measurement tools (distance/angle/area) are tier-gated (`createMeasurement` line 145), annotations are free.

## 2. Industry-standard benchmark   (3–5 lines; link ../research/imaging-ceph.md)

Per [research/imaging-ceph.md](../research/imaging-ceph.md): the clinical baseline is *multi-analysis* — Steiner, Downs, Ricketts, McNamara, Tweed, Jarabak, and the Wits appraisal — each with population norms (mean±SD, ethnicity-selectable). Modern competitors (WebCeph, AudaxCeph, Ceph Assistant, Dolphin) ship **fully automatic AI landmark detection** with manual correction, plus growth projection / VTO simulation. Calibration via a known-distance ruler (mm-per-pixel) is *mandatory* for valid linear measures. The ABO standard requires **three structural superimpositions** (cranial-base, maxillary, mandibular) registered on anatomical outlines — a single global align is below standard. FMX is a defined anatomical mount of PAs + bitewings; DICOM+PACS is the interoperability expectation (pano files ~50 MB).

## 4. UX/UI assessment   (Nielsen + WCAG; precision-pointing/landmark-drag ergonomics, keyboard access, error prevention on calibration; DESIGN.md fidelity)

- **Landmark ergonomics (strong).** Guided flow: next-unplaced landmark gets a `#FFE97D` ring + inline anatomical hint; selecting a placed (non-locked) landmark enables 1px arrow-key nudging; Tab/Enter advances (`ceph-keyboard.ts`); a magnifier loupe (`CephLoupe.tsx`) aids precision pointing. This matches Dolphin-style location-by-location entry well. Locked landmarks correctly become immutable (`disabled`), preventing accidental edits (Nielsen: error prevention).
- **Keyboard access (good for ceph, gap for measurement).** Ceph landmark placement is fully keyboard-drivable with `preventDefault` focus-trapping. The generic measurement toolbar and image transforms (rotate/flip/brightness via mouse buttons + range sliders) have no documented keyboard shortcuts beyond native; toolbar buttons use `aria-pressed`, which is good.
- **Calibration error prevention (partial).** Dialog enforces positive numeric mm (`disabled` until `parseFloat>0`), autofocus, Enter-to-confirm — good. **But the distance tool is not gated on calibration**: an uncalibrated measure silently renders in `px` (`canvas-overlays.tsx:24 ?? 'px'`). A clinician can read a "measurement" that is not in mm. Ceph linear metrics *are* hard-gated (422), so the risk is confined to the free-measure tool, but the toolbar's green "Calibrated" badge is the only signal — no warning when measuring uncalibrated. Recommend disabling distance/area until calibrated, or a persistent "uncalibrated — px only" banner.
- **Norm display (excellent, safety-conscious).** Amber/red-only chips (no green) with signed delta and `title` showing `mean±sd (source)` — avoids color-only reliance (WCAG 1.4.1) and avoids training clinicians to pattern-match green. Footnotes disclose ~7–13% ceph magnification, SN-vs-Frankfort non-interchangeability, and "reference ranges, not a diagnosis." Skeletal-pattern read-out is explicitly labeled "Informational — confirm clinically; not a diagnosis."
- **DESIGN.md fidelity.** Dark workspace (zinc-900/950) with `#FFE97D` lemon accent throughout (palette ring, badges, primary buttons) — consistent with the Apple-HIG + lemon system. Color contrast on amber-300/red-300 against zinc-800 chips should be spot-checked against WCAG AA (small text), but values are present alongside color.
- **Comparison UX.** "Compare ▶" appears only when exactly 2 images selected (clear affordance), offline placeholder handled gracefully. But it is two independent viewers — no synchronized zoom/pan, no overlay; users expecting before/after registration will be surprised (matches the documented deferral, but the term "Compare" oversells vs side-by-side).

## 5. Findings (P0–P3)

- **[P1] Only 2 of the 6–7 standard ceph analyses** — Steiner-SN + Ricketts only; Downs, McNamara, Tweed, Jarabak missing, Wits not offered. evidence: `ceph-math/src/index.ts` `ANALYSIS_TYPES`, `CephWorkspacePanel.tsx:42`. recommendation: prioritize McNamara (correction-magnitude mapping) + Tweed (already have IMPA/FMA-adjacent) + Wits as a single supplemental metric (AO-BO projection on occlusal plane) — the landmark set already supports Tweed/Wits; add per-analysis norm tables keyed like the existing two.
- **[P1] No DICOM ingest or pixel-spacing parsing** — MIME allowlist excludes `application/dicom`; `dicomMetadata` stores only filename; DICOM-tag calibration deferred. evidence: `imaging.schema.ts:124`, `createImagingStudy.ts:96`. recommendation: accept `application/dicom`, parse (0028,0030) PixelSpacing to auto-populate `pixelSpacingMm` with `calibration_method='dicom_tag'` (the enum value already exists), and size storage/transmission for ~50 MB pano/CBCT payloads (current 100 MB cap is OK, but no chunked/resumable handling).
- **[P1] Uncalibrated free-measure renders in px without a block/warning** — distance/area tools stay enabled and label in `px` when no calibration. evidence: `canvas-overlays.tsx:24` (`?? 'px'`), `measurement-toolbar.tsx` (no disable). recommendation: disable distance/area until calibrated (mirror the ceph 422 gate on the client), or show a persistent "Uncalibrated — values in px" alert. Note ceph mm-metrics are already correctly hard-gated.
- **[P1] No auto/AI landmarking** — fully manual; `ai`/`ai_corrected` provenance values exist but unused; flag off. evidence: `imaging_ceph.schema.ts:28`, MODULE_SPEC §18. recommendation: even a server-side heuristic seed (or batch import of detected points via the existing `batchUpsertCephLandmarks` with `source='ai'`) would close most of the gap; surface confidence and require clinician confirm (angular reliable, linear/soft-tissue weak per research).
- **[P1] Superimposition is side-by-side only** — no overlay, no cranial-base/maxillary/mandibular registration; below ABO standard. evidence: `comparison-view.tsx`, `CephReportView.tsx:279`. recommendation: implement an overlay mode with at least cranial-base (S-N) registration + opacity/onion-skin; this is the most clinically valuable timeline feature (see §6).
- **[P2] No FMX anatomical mount** — flat list, not a mounted PA+BW template. evidence: `patient-image-list.tsx`. recommendation: add an FMX mount layout component for intraoral series.
- **[P2] Single hardcoded norm population** — no ethnicity/age selector despite research showing population-specific norms. evidence: `ceph-math/src/norms.ts`. recommendation: extend `CEPH_NORMS` to a `{analysisType}.{population}` keying; UI selector.
- **[P2] CBCT/per-modality geometry not differentiated** — cbct enum present but treated as flat 2-D; only a panoramic distortion warning exists. evidence: `imaging.schema.ts:32`, `measurement-toolbar.tsx:57`. recommendation: scope explicitly (CBCT likely out of MVP) and label so users aren't misled.
- **[P3] No VTO/growth projection** — explicitly deferred (informational pattern read-out only). evidence: `CephReportView.tsx:284`. recommendation: keep deferred; the out-of-scope block (D-O) already discloses it honestly.

Note: the dedicated route `imaging-ceph-report.$imageId` (rendering `CephReportView`) could not be exercised live — no seeded ceph image exists for the test patient. Findings on the report view are from source reading (D-O out-of-scope block, calibration provenance line, immutable versioning) and remain unverified at runtime.

## 6. Carousel implications   (imaging superimposition/before-after over time is the imaging analog of our snapshot/carousel — how should imaging feed the timeline?)

Imaging is the clinical analog of the chart snapshot/carousel: a cephalometric **report version** (`imaging_ceph_report`, immutable append-only, D-I) is a snapshot exactly like a chart baseline, and serial superimposition is the imaging equivalent of stepping through carousel states. Three concrete ties:

1. **Report versions are already snapshots.** Each `createCephReport` mints an immutable, versioned, calibration-stamped snapshot. The timeline should surface these as discrete points (vN, date, calibration provenance) so a clinician scrubs through ceph states the way the carousel scrubs chart states — the data model is ready; only a timeline surface is missing.
2. **Findings + analyses should emit into the cumulative timeline.** Findings are audited and visit-linked (`imaging_finding.visitId/patientId`); they should feed the same cumulative snapshot model the carousel uses, so "what changed between visit A and B" includes new caries/periapical findings *and* ceph deltas (ANB, convexity) side-by-side with chart deltas.
3. **Superimposition is the imaging carousel "diff" — currently the biggest gap.** The before/after comparison is the visual diff between two timeline points. Today it is two unsynchronized viewers. To match the snapshot-first model, comparison should consume two report-version snapshots, register on a stable structure (start with cranial-base S-N, the existing reference frame), and render an overlay with opacity/onion-skin — turning the imaging timeline into a true superimposition carousel rather than a pair of static images. This is the single highest-leverage move to make imaging a first-class citizen of the snapshot/carousel timeline.
