# Per-type required-layer policy (coverage-completeness Phase 2)

**North star:** a user must never hit a missing or broken workflow. This policy
defines, for every authoritative surface item, *which test layers must prove it* —
so a gap can be detected, tracked, and ratcheted shut.

## The four layers

| Layer | What proves it | Source of truth |
|-------|----------------|-----------------|
| `be-unit` | a Bun test exercises the handler / business rule (direct call or via `buildTestApp`) | `services/api-ts/**/*.test.ts` (operationId-grep) |
| `contract` | a Hurl request asserts the wire contract | `specs/api/tests/contract/**.hurl` (`endpoint-matrix`) |
| `fe-unit` | a React/RTL test exercises the component/hook | `apps/dentalemon/src/**/*.test.tsx` |
| `e2e` | a Playwright journey/spec drives the workflow end-to-end | `apps/dentalemon/tests/e2e/**` (`workflow-matrix` + spec-index) |

## Required-layer matrix (by item type × risk)

Risk classes: **money** = `billing`, `dental-billing`; **clinical/safety** =
`dental-clinical`, `dental-visit`, `dental-perio`, `dental-imaging`, `dental-pmd`,
`dental-patient`, `dental-erasure`, `dental-legalhold`, `retention`, `emr`, `audit`,
`person`, `patient` (PII/compliance); otherwise ordinary.

| Item | be-unit | contract | fe-unit | e2e |
|------|:------:|:--------:|:-------:|:---:|
| User-reachable **workflow** (`type=workflow`/`inter-module`, has route) | ✅ if it has an endpoint | ✅ if money/clinical | ✅ | ✅ |
| Money/clinical user-facing **mutation** (`use-case` write, has route) | ✅ | ✅ | ✅ | ✅ |
| Ordinary **write** (`use-case`, has route) | ✅ | — | ✅ | — |
| **Read** (`use-case` read, has endpoint) | ✅ | — | — | — |
| **Business rule** (`type=business-rule`) | ✅ | ✅ if money/clinical & has endpoint | — | — |
| Backend-only / orphan endpoint (no FE consumer) | governed by `endpoint-matrix` orphan ratchet, **not** the user-workflow backlog |

This is exactly what `scripts/coverage/ledger-coverage.ts` `policyLayers()` computes.
The plan's decision 3 in one line: *every user-reachable workflow needs an e2e proof;
money/clinical/inter-module get the full stack; ordinary writes get be+fe unit; reads
get be-unit.*

## Artifacts & how they relate

| File | What it is | Regenerated by |
|------|-----------|----------------|
| `coverage-ledger.json` | the authoritative surface (915 items, Phase 0) | the Phase 0 reader workflow |
| `ledger-coverage.json` | raw deterministic per-item layer grades (Phase 1 engine) | `bun run coverage:ledger` |
| `coverage-backlog.json` | the **curated, LLM-verified** gap list to burn down (Phase 3) | Phase 1 verify workflow |
| `ledger.allowlist.json` | shrink-only baseline of tolerated user-reachable gaps | hand-maintained (seeded Phase 2) |

`ledger-coverage.json` is deterministic and CI-reproducible (operationId-grep for
be-unit, static spec-index for e2e — **no test run required**). Its raw view has
known false positives (grep is blind to `buildTestApp` route-only be-unit tests; the
e2e spec-index is a fuzzy testid/route match). `coverage-backlog.json` is the curated
truth — each entry confirmed real by reading the corpus.

## Enforcement (the ratchets)

| Gate | Scope | Mode |
|------|-------|------|
| `coverage:workflows` (`workflow-matrix`) | 108 catalogued WF-* (now incl. `[INFERRED]`) → journey/e2e | RATCHET (`workflow.allowlist.json`, shrink-only) |
| `coverage:endpoints` (`endpoint-matrix`) | 380 endpoints → contract/integration/journey | RATCHET (`endpoint.allowlist.json`) |
| `coverage:roles` (`role-op-matrix`) | role↔op authz | HARD (never allowlisted) |
| `coverage:fsm`, `coverage:fe-route` | status transitions, FE routes | RATCHET |
| `coverage:br` (`br-matrix`) | business rules → proving test | REPORT-ONLY |
| **`coverage:ledger:check`** (`ledger-coverage`) | **915-item ledger, user-reachable gaps** | **REPORT-ONLY** (advisory) |
| journey CORE gate (`run-journey-harness`) | the 9 core workflows | HARD (SKIP fails) |

**Why the ledger ratchet is report-only (for now):** the deterministic gap detection
has false positives a hard gate would fail spuriously on legitimately-tested new code.
Promote to blocking once detection is hardened (e.g. the coverage recorder runs in CI
to fill the be-unit signal) **and** branch-protection enforcement is enabled (currently
blocked on a GitHub plan/visibility decision). Until then it runs in CI and reports
new user-reachable gaps for reviewer attention.

## RED-first for new work

Per [VERTICAL_TDD.md](../../development/VERTICAL_TDD.md): a new workflow ships with its
required layers *test-first*. A new user-reachable gap that appears in
`coverage:ledger:check` must be closed RED→GREEN (Phase 3 pattern) or, if genuinely
deferred, added to `ledger.allowlist.json` with a reason — the allowlist only ever
shrinks by an explicit, reviewed edit.
