## Critical Gaps

### BFIX-01 — Price Save BigInt Crash (BLOCKER)

**File:** `apps/dentalemon/src/features/workspace/hooks/use-save-treatment.ts:36`

**Root cause:** Commit `1f6178d` fixed WR-06 (removed BigInt). Commit `91c91bd` reverted it ("restore BigInt for SDK priceCents type compatibility"). Current code:
```ts
priceCents: BigInt(Math.round(priceAmount * 100))
```
`JSON.stringify()` cannot serialize `BigInt` — throws `TypeError: Do not know how to serialize a BigInt`. This crashes the tooth slideout save flow.

**Affected flows:** Tooth slideout → add/edit treatment → save

**Note:** The treatment-table inline edit path (`useUpdateTreatment`) is unaffected — it never used BigInt.

**Fix options:**
1. Fix SDK generated type: change `priceCents` from `bigint` to `number` in `packages/sdk-ts` (requires SDK regen after TypeSpec change)
2. Immediate workaround: cast at call site — `priceCents: Number(Math.round(priceAmount * 100))`
