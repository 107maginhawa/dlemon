## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (`px-1.5 py-0.5`) |
| sm | 8px | Compact element spacing, button gap (`gap-2`) |
| md | 16px | Default element spacing, sheet padding (`px-4`) |
| xl | 24px | Sheet header/footer padding (`px-6 py-4`), section padding |
| 2xl | 32px | Major layout gaps |

Exceptions:
- Touch targets (buttons in table rows): minimum 44px height — use `h-11` (44px) for all primary action buttons

Source: Measured from `consent-sheet.tsx` and `treatment-table.tsx`.

## Component Contracts

### 1. Inline Price Edit (TXTBL-02)

**Trigger:** Click on price cell (`<td>`) in TreatmentTable — non-read-only visit only.

**Edit mode visual:**
- Replace static `CURRENCY_SYMBOL + amount` text with `<input type="number">` in-place
- Input width: `w-20`
- Alignment: `text-right`
- Border: `border border-border rounded px-1`
- Size: `text-sm`
- `autoFocus` on mount

**Save behavior:**
- `onBlur` fires → multiply display value × 100 → send `priceCents` to `updateDentalTreatmentMutation`
- Guard: if `parseFloat(draftPrice)` is NaN or ≤ 0, abort mutation, revert to previous value
- On success: invalidate `listDentalTreatmentsQueryKey`

**Read-only gate:** When `readOnly={true}`, price cell is non-clickable static text. No hover state.

**Currency display:** `CURRENCY_SYMBOL` + `value.toLocaleString(APP_LOCALE)` — from `@/constants/brand`.

### 2. Dismiss Treatment Popover (TXTBL-03)

**Trigger:** "Dismiss" action — rendered as a text button `text-xs text-destructive hover:underline` inside the treatment row (same column as "Mark Done").

**Popover spec:**
- Component: `@radix-ui/react-popover` (`Popover`, `PopoverTrigger`, `PopoverContent`)
- Width: `w-64`
- Padding: `p-4`
- `side="left"` preferred (avoids overflow off-screen on table right edge); fall back to `side="bottom"`

**Popover content:**
- Label: `text-xs font-semibold` — "Reason (required)"
- Input: full-width `<input type="text">`, `border rounded px-2 py-1 text-sm`, `minLength={3}`, placeholder: "e.g. Patient declined"
- Confirm button:
  - Copy: "Confirm Dismiss"
  - Style: `mt-2 w-full rounded bg-destructive/10 text-destructive text-sm py-1.5 font-medium disabled:opacity-50`
  - Disabled when `reason.trim().length < 3`
- On confirm: `updateDentalTreatmentMutation({ path: { visitId, treatmentId }, body: { status: 'dismissed', dismissReason: reason } })`
- On success: close popover + invalidate `listDentalTreatmentsQueryKey`

**Read-only gate:** Dismiss button not rendered when `readOnly={true}` or `t.status === 'dismissed'`.

### 3. Inline Notes Sub-Row (TXTBL-04)

**Trigger:** Chevron button (`ChevronDown` / `ChevronUp` from lucide-react) in a new "Notes" column at far-left or appended after Status column.

**Collapsed state:** Chevron right (`ChevronRight`, 14px, `text-muted-foreground`).

**Expanded state:**
- Sub-row spans all 7 columns (`colSpan={7}`)
- Background: `bg-muted/30`
- Padding: `px-4 py-2`
- Content: `<textarea>` — `w-full text-sm border border-border rounded px-2 py-1.5 resize-none rows={2}`, placeholder: "Add notes…"
- Local state only — no API call (backend has no notes field on treatment schema)
- Notes persist only in component state during the session

**Read-only gate:** When `readOnly={true}`, chevron still reveals the sub-row but textarea is `readOnly` and styled `opacity-70 cursor-default`.

### 4. Toggle Completed Treatments (TXTBL-05)

**Button:** Existing "View Completed (N)" button — already rendered in `TreatmentTable` header area.

**Behavior:**
- Local `useState<boolean>(false)` — `showCompleted` — in `TreatmentTable`
- When `showCompleted === false`: filter out rows where `t.status === 'performed' || t.status === 'verified'` (the "completed" statuses in the real backend enum)
- Button copy: "View Completed ({count})" when hidden; "Hide Completed ({count})" when shown
- Button style: `text-xs text-muted-foreground hover:text-foreground` — matches existing

### 5. Dual Subtotals (TXTBL-01)

**Location:** Two rows immediately above the existing "Grand Total" row in `<tbody>`.

**Row structure (This Visit):**
```
<tr data-testid="subtotal-this-visit-row" className="border-t border-border/40">
  <td colSpan={6} className="px-4 py-1.5 text-right text-xs text-muted-foreground">This Visit</td>
  <td className="px-4 py-1.5 text-right tabular-nums text-xs text-muted-foreground">
    {CURRENCY_SYMBOL}{thisVisitTotal.toLocaleString(APP_LOCALE)}
  </td>
</tr>
```

**Row structure (Carried Over):** same structure, `data-testid="subtotal-carried-over-row"`, label "Carried Over".

**Grand Total row:** Unchanged (`border-t-2 border-border font-semibold text-sm`) — stays as-is.

**Computation:**
- `thisVisitTotal = treatments.reduce((sum, t) => sum + (t.priceAmount ?? 0), 0)`
- `carriedOverTotal = carriedOverItems.reduce((sum, i) => sum + (i.priceCents / 100), 0)`
- Only render carried-over subtotal row when `carriedOverItems.length > 0`

### 6. SoapNotesSheet (VISIT-04)

**Pattern:** Mirror `consent-sheet.tsx` exactly — `fixed inset-0 z-40 flex items-end`, backdrop, slide-up panel.

**Panel dimensions:** `max-h-[80vh] bg-background rounded-t-2xl shadow-2xl flex flex-col`

**Header:**
- Drag handle: `w-9 h-1 bg-muted-foreground/30 rounded-full` centered at top
- Title: "SOAP Notes" — `text-base font-semibold`
- Close button: `w-7 h-7 rounded-full bg-secondary` with ✕

**Body (scrollable, `flex-1 overflow-y-auto px-6 py-4 flex flex-col gap-4`):**

| Field | Label | Type | Placeholder |
|-------|-------|------|-------------|
| Subjective | "Subjective" | `<textarea rows={3}>` | "Chief complaint, patient-reported symptoms…" |
| Objective | "Objective" | `<textarea rows={3}>` | "Clinical findings, vitals, exam results…" |
| Assessment | "Assessment" | `<textarea rows={2}>` | "Diagnosis, clinical impression…" |
| Plan | "Plan" | `<textarea rows={3}>` | "Treatment planned, referrals, follow-up…" |
| Notes | "Additional Notes" | `<textarea rows={2}>` | "Any other relevant notes…" |

Field label style: `text-xs font-semibold text-muted-foreground uppercase tracking-wide mb-1.5 block`

Textarea style: `w-full rounded-xl border border-border px-3 py-2 text-sm bg-background focus:border-[#FFE97D] outline-none resize-none`

**Loading state:** Skeleton rows (3 of them, `h-16 bg-muted animate-pulse rounded-xl`) while `getVisitNotesOptions` query is pending.

**Footer (sticky, `flex gap-2 px-6 py-4 border-t flex-shrink-0`):**
- Discard: `flex-1 h-11 rounded-xl border border-border text-sm hover:bg-secondary`
- Save Notes: `flex-1 h-11 rounded-xl bg-[#FFE97D] text-[#4A4018] text-sm font-semibold hover:bg-[#F5DC60] disabled:opacity-50`
  - Disabled while `saving === true`
  - Copy: "Save Notes" (idle) → "Saving…" (mutating)

**Data flow:**
- On open: `useQuery(getVisitNotesOptions({ path: { visitId } }))` pre-populates fields
- Save: `useMutation({ ...upsertVisitNotesMutation() })` → on success invalidate `getVisitNotesOptions`
- Sheet replaces existing `MedicalHistoryForm` inside `notesSheetOpen` — add "View Medical History" secondary link below the Notes textarea

**Landmine:** `$patientId.tsx` line 341 currently shows `MedicalHistoryForm` on `notesSheetOpen`. This component replaces that. Add `<button className="text-xs text-muted-foreground hover:underline mt-2">View Medical History</button>` that opens `MedicalHistorySheet`.

### 7. PreCompletionChecklist Dialog (VISIT-02)

**Component:** `@radix-ui/react-dialog`

**Trigger:** "Complete Visit" button — location: workspace footer or WorkspaceTopBar (executor's discretion; WorkspaceTopBar preferred for consistency with `onNotes` pattern).

**Trigger button style:** `h-11 px-4 rounded-xl bg-[#FFE97D] text-[#4A4018] text-sm font-semibold hover:bg-[#F5DC60] disabled:opacity-50`

**Dialog dimensions:** `max-w-sm w-full mx-auto` — centered, `p-6`, `rounded-2xl`

**Header:**
- Title: "Complete Visit" — `text-base font-semibold`
- Subtitle: `text-xs text-muted-foreground` — "Checking visit readiness…" (loading) / "Review the items below." (loaded)

**Loading state:** 4 skeleton rows — `h-8 bg-muted animate-pulse rounded-lg mb-2`

**Loaded state — check rows:**

Each row: `flex items-center gap-3 py-2 border-b border-border/40 last:border-0`

| Check | Pass Icon | Warn Icon | Pass Condition |
|-------|-----------|-----------|----------------|
| Consent signed | `CheckCircle2` size-4 `text-[#34C759]` | `AlertTriangle` size-4 `text-[#FF9500]` | At least 1 signed consent form exists |
| No unstarted treatments | `CheckCircle2` `text-[#34C759]` | `AlertTriangle` `text-[#FF9500]` | No treatments with `status === 'diagnosed' || 'planned'` |
| SOAP notes present | `CheckCircle2` `text-[#34C759]` | `AlertTriangle` `text-[#FF9500]` | `getVisitNotes` returns a record with at least one non-empty field |
| No open lab orders | `CheckCircle2` `text-[#34C759]` | `AlertTriangle` `text-[#FF9500]` | No lab orders with `status !== 'completed' && status !== 'cancelled'` |

Check label: `text-sm` — e.g. "Consent signed", "No unstarted treatments", "SOAP notes present", "No open lab orders"

Sub-label (warn only): `text-xs text-muted-foreground` — e.g. "2 treatments still planned", "SOAP notes not recorded"

**Footer (after checks load):**
- Go Back: `flex-1 h-11 rounded-xl border border-border text-sm hover:bg-secondary`
- Complete Visit: `flex-1 h-11 rounded-xl text-sm font-semibold`
  - All pass: `bg-[#FFE97D] text-[#4A4018] hover:bg-[#F5DC60]`
  - Any warn: `bg-destructive/10 text-destructive hover:bg-destructive/20`
  - Copy when any warn: "Complete anyway"
  - Copy when all pass: "Complete Visit"

**On confirm:** `updateDentalVisitMutation({ path: { visitId }, body: { status: 'completed' } })` → invalidate `listDentalVisitsQueryKey`

**Disabled state:** "Complete Visit" trigger button is `disabled` when `currentVisit.status !== 'active'`.

### 8. Lock Visit Button (VISIT-03)

**Location:** Inside each completed visit card in `TimelineCarousel`.

**Appearance:**
- Only visible when `card.status === 'completed'`
- Style: `h-8 px-3 rounded-lg border border-border text-xs font-medium text-muted-foreground hover:text-foreground hover:border-foreground/50 transition-colors`
- Icon: `Lock` from lucide-react, size-3, `mr-1`
- Copy: "Lock Visit"

**Confirmation pattern:**
- Single-click on the Lock button → shows `window.confirm("Lock this visit? Locked visits cannot be edited.")` — acceptable for v1.2.1; no additional dialog needed (keep it lightweight)
- On confirm: `updateDentalVisitMutation({ path: { visitId: card.id }, body: { status: 'locked' } })` → invalidate visits query
- On cancel: no action

**Locked state:**
- Card renders with `opacity-90` tint
- A `Lock` icon size-3 `text-muted-foreground` appears next to the visit date
- No Lock button shown
- All treatment rows in TreatmentTable are `readOnly={true}`
- "Complete Visit" and all edit/dismiss/price-edit actions suppressed

## Read-Only Propagation

`readOnly` flag derives from: `visit.status === 'completed' || visit.status === 'locked'`

Pass as `readOnly={isReadOnly}` to `TreatmentTable`. When `readOnly`:
- Price cells: non-clickable, no hover state
- Dismiss buttons: not rendered
- Mark Done buttons: not rendered
- Notes sub-row textarea: `readOnly` attribute, styled `opacity-70 cursor-default`
- "Complete Visit" button: not rendered (visit is already completed or locked)

## Accessibility

- All interactive elements have `aria-label`
- Sheet overlays: `role="dialog" aria-modal="true"`
- Dialog: use Radix `Dialog.Root` which handles focus trap automatically
- Popover: use Radix `Popover.Root` which handles escape-to-close
- Color is not the sole differentiator for pass/warn states — icon shape distinguishes them (CheckCircle2 vs AlertTriangle)
- Price input: `aria-label="Edit price for treatment {description}"`
- Dismiss confirm button: `aria-label="Confirm dismiss treatment"`
