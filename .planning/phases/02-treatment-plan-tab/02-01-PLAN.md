# Phase 2 Plan: Treatment Plan Tab

**Phase Goal**: Replace "coming in PR2" placeholder with live treatment plan data, grouped by urgency (status), with total cost summary.

**Requirements**: TXPL-01, TXPL-02, TXPL-03

---

## Context

### Current State
- `$patientId.tsx` ‚Äî `treatment-plan` tab falls through to catch-all: `{activeTab.replace('-', ' ')} ‚Äî coming in PR2`
- `app.ts:140` ‚Äî `GET /dental/patients/:patientId/treatment-plan` already registered with `dentalAuth`
- No frontend hook for patient-level treatment plan (only visit-level `useTreatments` hitting `/dental/visits/:visitId/treatments`)

### API Response Shape
```ts
{
  patientId: string;
  totalEstimateCents: number;    // TXPL-03: total cost
  treatmentCount: number;
  toothCount: number;
  byTooth: Record<string | 'general', TreatmentPlanItem[]>;
  treatments: TreatmentPlanItem[];
}

interface TreatmentPlanItem {
  id: string;
  toothNumber: number | null;
  cdtCode: string;
  description: string;
  surfaces: string[] | null;
  priceCents: number;
  status: 'diagnosed' | 'planned';  // TXPL-02: urgency grouping
  conditionCode: string | null;
  visitId: string;
  carriedOver: boolean;
}
```

### Urgency Grouping (TXPL-02)
- `diagnosed` ‚Üí "Diagnosed" section (condition found, treatment recommended immediately)
- `planned` ‚Üí "Planned" section (treatment scheduled for future appointment)

## Tasks

### Task 1: Hook ‚Äî `use-treatment-plan.ts`
**File**: `apps/dentalemon/src/features/workspace/hooks/use-treatment-plan.ts`

- TanStack Query hook, `queryKey: ['dental-treatment-plan', patientId]`

### Task 2: Component ‚Äî `treatment-plan-tab.tsx`
**File**: `apps/dentalemon/src/features/workspace/components/treatment-plan-tab.tsx`

Props:
```ts
interface TreatmentPlanTabProps {
  patientId: string;
}
```

Sections (in render order):
1. **Loading state** ‚Äî skeleton rows while `isLoading`
2. **Empty state** ‚Äî "No pending treatments" message when `treatments.length === 0`
3. **Diagnosed section** (TXPL-02) ‚Äî `data-testid="group-diagnosed"`, rendered if any `status === 'diagnosed'`
4. **Planned section** (TXPL-02) ‚Äî `data-testid="group-planned"`, rendered if any `status === 'planned'`
5. **Cost summary** (TXPL-03) ‚Äî `data-testid="cost-summary"`, shows `totalEstimateCents / 100` formatted with `CURRENCY_SYMBOL` + `APP_LOCALE`, treatment count, tooth count

Each treatment row (`data-testid="treatment-row"`):
- Tooth# (or "General" for null)
- CDT code badge
- Description
- Surfaces (if any) ‚Äî comma-joined
- Price: `(priceCents / 100).toLocaleString(APP_LOCALE, { style: 'currency', currency: 'USD' })`

Design: Tailwind + BRAND_GOLD accent, consistent with workspace design system (lemon `#FFE97D`, `BRAND_GOLD_TEXT` `#4A4018`).
