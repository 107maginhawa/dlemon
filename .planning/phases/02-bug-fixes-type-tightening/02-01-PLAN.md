### Task 1: Tighten dental-chart.helpers.ts types (BUG-04, BUG-05)

<read_first>
- apps/dentalemon/src/features/workspace/components/dental-chart.helpers.ts
- apps/dentalemon/src/features/workspace/components/five-surface-selector.helpers.ts
</read_first>

<action>
In `apps/dentalemon/src/features/workspace/components/dental-chart.helpers.ts`:

1. Add import at top:
   ```typescript
   import type { ToothSurface } from '@/features/workspace/components/five-surface-selector.helpers';
   ```

2. Change `ToothData.surfaces` from `string[]` to `ToothSurface[]`:
   ```typescript
   export interface ToothData {
     toothNumber: number;
     state: ToothState;
     surfaces?: ToothSurface[];   // was: string[]
     conditionCode?: string;
     note?: string;
   }
   ```

3. Tighten `buildToothMap` signature — parameter type `state: string` → `state: ToothState`, return type `Map<number, string>` → `Map<number, ToothState>`:
   ```typescript
   export function buildToothMap(teeth: Array<{ toothNumber: number; state: ToothState }>): Map<number, ToothState> {
     const map = new Map<number, ToothState>();
     for (const tooth of teeth) {
       map.set(tooth.toothNumber, tooth.state);
     }
     return map;
   }
   ```

4. Tighten `getToothColorClass` — remove `| string` from parameter:
   ```typescript
   export function getToothColorClass(state: ToothState): string {
   ```
   (The function body handles all ToothState values; | string was unnecessary widening.)

5. Add FDI guard functions after the `TOOTH_NUMBERS` const (after line 32):
   ```typescript
   /**
    * Returns true if n is a valid FDI permanent tooth number (11–18, 21–28, 31–38, 41–48).
    */
   export function isValidFdiNumber(n: number): boolean {
     return TOOTH_NUMBERS.includes(n);
   }

   /**
    * Returns true if n is a valid Universal tooth number (1–32).
    */
   export function isValidUniversalNumber(n: number): boolean {
     return Number.isInteger(n) && n >= 1 && n <= 32;
   }
   ```

After edits, verify all call sites of `getToothColorClass` and `buildToothMap` in the file compile — they should since the switch statement already covers all ToothState values.
</action>
