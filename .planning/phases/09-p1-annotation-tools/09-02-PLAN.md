# Plan 09-02: Frontend — Annotation Toolbar + SVG Rendering

## Objective

Add AnnotationToolbar component (Label|Arrow|Freehand|Shape|Tooth buttons), extend `ToolMode`
type and SVG rendering in `imaging-workspace.tsx` for all 5 annotation types, and write a
Playwright E2E spec.

## Context

- D5: Extend existing SVG overlay from Phase 8
- D6: AnnotationToolbar is a separate component parallel to MeasurementToolbar
- D7: SVG rendering: label→<text>, arrow→<line>+arrowhead, freehand→<polyline>, shape→<rect>/<ellipse>, tooth→<circle>+<text>
- D8: invalidateQueries after mutation (last-write-wins, BR-035)
- D9: Playwright spec instead of human checkpoint

## Tasks

### Task 1: Extend ToolMode type in measurement-toolbar.tsx

Extend `ToolMode` export in `measurement-toolbar.tsx`:
```typescript
export type ToolMode = 'none' | 'calibration' | 'distance' | 'angle' | 'area' | 'label' | 'arrow' | 'freehand' | 'shape' | 'tooth'
```

### Task 2: Create annotation-toolbar.tsx

Create `apps/dentalemon/src/features/imaging/components/annotation-toolbar.tsx`:
- Same pattern as `measurement-toolbar.tsx`
- Buttons: Label | Arrow | Freehand | Shape | Tooth
- Each button toggles its mode (click active → 'none')
- `aria-pressed` attribute on each button
- Visual: bg-zinc-800 border-b border-zinc-700 to match MeasurementToolbar
- Export: `AnnotationToolbar` component + `ANNOTATION_TOOLS` array

### Task 3: Extend imaging-workspace.tsx

1. Import `AnnotationToolbar` and render it below `MeasurementToolbar`

2. Add `AnnotationShape` sub-component for rendering saved annotations:
   - `label` → `<text>` at `geometry.point.x, geometry.point.y` with `geometry.text`
   - `arrow` → `<line>` from→to with `marker-end` referencing a `<defs><marker>` arrowhead
   - `freehand` → `<polyline>` from `geometry.points` array
   - `shape` rect → `<rect>` at geometry.x,y with geometry.width,height
   - `shape` ellipse → `<ellipse>` with cx/cy derived from x+width/2, y+height/2, rx=width/2, ry=height/2
   - `tooth` → `<circle>` at geometry.point + `<text>` with toothNumber
   - All have a delete circle (red, r=6) like MeasurementShape

3. Extend `DrawingPreview` for annotation modes:
   - `label` / `tooth` → single dot at first point
   - `arrow` → dashed line from→to (2 clicks)
   - `freehand` → polyline (accumulates clicks; double-click to finish)
   - `shape` → dashed rect preview (2 clicks: top-left, bottom-right)

4. Extend `handleSvgClick` for annotation tool modes:
   - `label` / `tooth` → 1 click → open inline prompt (use `window.prompt` for text/toothNumber) → mutate → reset
   - `arrow` → 2 clicks → mutate → reset
   - `freehand` → accumulate points, double-click to finish → mutate → reset
   - `shape` → 2 clicks (top-left + bottom-right) → mutate → reset
   - All call `createMeasurement.mutate({ type: mode, geometry: {...} })`

5. Add `<defs>` with arrowhead `<marker>` inside the SVG for arrow rendering

### Task 4: Write Playwright E2E spec

Create `apps/dentalemon/tests/e2e/imaging-annotation.spec.ts` following the pattern of
`imaging-measurement.spec.ts`:
- Tests for AnnotationToolbar rendering all 5 buttons
- Test aria-pressed toggling for each annotation tool
- Test annotation toolbar is separate from measurement toolbar
- Test SVG overlay data-testid is present
- Test annotation tool activates crosshair cursor on SVG
