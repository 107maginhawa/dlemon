# Plan 09-01: Backend — Extend Annotation Geometry Types

## Objective

Extend the Zod discriminated union in `createMeasurement.ts` with 5 annotation geometry types
(label, arrow, freehand, shape, tooth). Add tests for all 5 types. No new DB tables or TypeSpec
ops needed — reuses Phase 8 endpoints.

## Context

- `createMeasurement.ts` already has distance/angle/area in its Zod union
- `imaging_annotation` table already has all 5 annotation types in `imagingAnnotationTypeEnum`
- D3: No tier gate on annotations — any tier can create them
- BR-023: geometry stored as JSON, never burned into image

## Tasks

### Task 1: Extend Zod union with 5 annotation geometry types

Extend `createMeasurement.ts`:

1. Add 5 new geometry schemas after existing AreaGeometry:
   - `LabelGeometry`: `{ type: 'label', point: {x,y}, text: string(1-200) }`
   - `ArrowGeometry`: `{ type: 'arrow', from: {x,y}, to: {x,y} }`
   - `FreehandGeometry`: `{ type: 'freehand', points: [{x,y}].min(2) }`
   - `ShapeGeometry`: `{ type: 'shape', shapeType: enum('rect','ellipse'), x,y,width,height: number }`
   - `ToothGeometry`: `{ type: 'tooth', point: {x,y}, toothNumber: int.min(1).max(32) }`

2. Extend `MeasurementGeometry` discriminated union to include all 5 new types

3. Extend `TYPE_MAP` to map new types to `imagingAnnotationTypeEnum` values:
   - `label` → `'label'`
   - `arrow` → `'arrow'`
   - `freehand` → `'freehand'`
   - `shape` → `'shape'`
   - `tooth` → `'tooth'`

4. Fix the tier gate: annotation types (label/arrow/freehand/shape/tooth) must bypass the
   `free` tier check (D3 — no tier gate on annotations). Only distance/angle/area are tier-gated.

5. Update type validation error message to include all valid types

### Task 2: Add tests for all 5 annotation types

Extend `imaging.test.ts` with a new `describe('annotation geometry types')` block covering:
- label: valid label at point → 201
- arrow: valid arrow from→to → 201
- freehand: valid freehand with 2+ points → 201
- shape rect: valid rect shape → 201
- shape ellipse: valid ellipse shape → 201
- tooth: valid tooth annotation (toothNumber 1-32) → 201
- tooth: invalid toothNumber (0 or 33) → 400
- label: empty text → 400
- freehand: only 1 point → 400
- annotation types on free tier → 201 (no tier gate)

## Verification

Run: `cd services/api-ts && bun test src/handlers/dental-imaging/imaging.test.ts`

All tests must pass (green).

## Success Criteria

- All 5 annotation geometry types validated correctly
- Invalid payloads return 400
- Annotations bypass free-tier gate
- All tests pass
