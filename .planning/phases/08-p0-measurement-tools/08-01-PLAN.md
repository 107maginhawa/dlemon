<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /dental/imaging/images/{imageId}/calibration | pixelSpacingMm float value from untrusted client; must be positive |
| client → /dental/imaging/images/{imageId}/measurements | geometry JSONB from untrusted client; coordinate injection risk |
| client → /dental/imaging/measurements/{measurementId} | path param; must verify ownership before delete |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | All 4 endpoints | mitigate | UnauthorizedError thrown if user.id absent (same as existing handlers) |
| T-08-02 | Tampering | createMeasurement geometry | mitigate | Zod discriminated union validates geometry shape before insert; ValidationError on parse failure |
| T-08-03 | Elevation of privilege | createMeasurement tier gate | mitigate | resolveImagingTier coerces null→'free'; 403 returned before any DB write |
| T-08-04 | Information disclosure | listMeasurements cross-branch | mitigate | assertBranchAccess validates user belongs to image's branchId (per D5) |
| T-08-05 | Tampering | updateImageCalibration float | mitigate | Validate pixelSpacingMm > 0 at handler level; reject non-positive values with 400 |
| T-08-06 | Elevation of privilege | deleteMeasurement ownership | mitigate | Fetch annotation first, verify branch membership before delete |
</threat_model>
