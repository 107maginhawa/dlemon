**getThumbnailPipClass(state: ToothState): string** — colocated in this file, NOT imported from dental-chart.helpers:
- `healthy` → `'bg-muted'`
- `caries` → `'bg-red-500/40'`
- `fractured` → `'bg-amber-400'`
- `filled` → `'bg-green-500'`
- `crown` → `'bg-blue-400'`
- `extracted` → `'border border-dashed border-red-500 bg-transparent'`
- `missing` → `'bg-muted/50'`
- `implant` → `'bg-blue-300'`
- `watchlist` → `'bg-amber-300'`

**Layout:**
- Build a Map<toothNumber, state> from the `teeth` prop for O(1) lookups.
- Upper jaw row: teeth 11,12,13,14,15,16,17,18,21,22,23,24,25,26,27,28 — render left to right.
- Lower jaw row: teeth 31,32,33,34,35,36,37,38,41,42,43,44,45,46,47,48 — render left to right.
- Container: `<div className="flex flex-col gap-0.5 px-3 pb-2" data-testid="dental-chart-thumbnail">`.
- Each row: `<div className="grid grid-cols-16 gap-px">`.
- Each pip: `<div className={`w-1 h-1 rounded-sm ${getThumbnailPipClass(state)}`} data-tooth={toothNumber} />`.
- If a tooth number is not in the `teeth` array, use `getThumbnailPipClass('healthy')` as default.
- Add `aria-label="Dental chart thumbnail"` on the container div.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API → RawPatient | Patient list JSON from backend crosses into frontend transform |
| RawPatient.status → tabClass | Unchecked string cast to union type drives UI color selection |
| RawPatient.latestChartTeeth[].state → ToothState | Unchecked string cast to union type drives pip color selection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | toPatientCard — status cast | accept | Status only affects tab color; no security gate. Invalid values fall through to `default: 'bg-lemon'` in tabClass switch. Low-value target. |
| T-03-02 | Tampering | toPatientCard — state cast to ToothState | accept | State only drives pip background color class. Worst case: unrecognized value hits no switch arm and returns fallback (healthy = bg-muted). No PII exposed. |
| T-03-03 | Information Disclosure | DentalChartThumbnail data-tooth attributes | accept | Tooth numbers are clinical, non-PII data already shown in full chart. Presence in DOM attributes adds no new disclosure surface. |
| T-03-04 | Elevation of Privilege | tabClass switch default | mitigate | Ensure TypeScript exhaustive: switch on union type. Add `default: return 'bg-lemon'` to prevent CSS injection if union expands without updating switch. |
</threat_model>
