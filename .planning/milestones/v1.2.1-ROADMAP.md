## Milestone Summary

### Key Decisions

- **Carousel snapshot model:** Cumulative snapshot approach, 6 visit states, snapshot-first
- **Visit lifecycle safety checks:** 4-check pre-completion checklist (not inline validation)
- **TypeSpec migration scope:** Backend-only (services/api-ts); SDK regen deferred to separate step
- **priceCents int32 (not int64):** DB uses PostgreSQL integer; int64 was incorrect (caused BigInt crash)
- **BFIX-02 accepted deviation:** useTreatmentPlan raw fetch accepted; no generated SDK hook existed

### Issues Resolved

- BigInt serialization crash (WR-06): root-caused to TypeSpec int64→int32 mismatch vs DB integer
- Reactive org store reads: .getState() in render paths causing stale data
- WorkspaceTabs orphan: deleted with zero import cleanup
- Fullscreen escape: event-driven via fullscreenchange listener
