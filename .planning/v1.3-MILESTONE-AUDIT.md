## Gaps Found

### Gap 1 — BLOCKER: `listPatientImages` route path wrong in generated routes.ts

**Severity:** BLOCKER — production bug; every image list call returns 404
**File:** `services/api-ts/src/generated/openapi/routes.ts` line 869

- **TypeSpec (source of truth):** `@route("/dental/patients/{patientId}/images")` in `specs/api/src/modules/dental-imaging.tsp` ✅
- **Generated route:** `app.get('/dental/patients/:patientId', ...)` ❌ — missing `/images` suffix
- **Frontend fetch:** `/dental/patients/${patientId}/images` (correct per TypeSpec) ✅

**Root cause:** Codegen did not propagate the full route path. `specs/api` needs rebuild + `services/api-ts` needs regeneration.

**Fix:**
```bash
cd specs/api && bun run build
cd ../../services/api-ts && bun run generate
# Verify: grep 'listPatientImages' src/generated/openapi/routes.ts
```

**Impact:** `PatientImageList` cannot load — entire imaging feature is non-functional in production.

### Gap 2 — KNOWN: E2E test harness routes absent

**Severity:** TEST COVERAGE (documented known gap from Phase 10)
**Spec files:** `tests/e2e/imaging-comparison.spec.ts`, `imaging-measurement.spec.ts`, `imaging-annotation.spec.ts`

Tests navigate to `/imaging-test` and `/imaging-comparison-test` routes that don't exist. Self-skip pattern applied — tests auto-skip when server URL unavailable. All 31 E2E tests are non-executable without a test harness route.

**Documented in:** Phase 10 verification (2026-05-11) as accepted design decision.

**Deferred to:** v1.4 or v1.5 (create `/imaging-test` harness routes for E2E execution).

### Gap 3 — MINOR: Missing `data-testid` attributes in imaging components

**Severity:** TEST QUALITY (affects E2E assertions, not production behavior)
**Files:**
- `apps/dentalemon/src/features/imaging/components/annotation-toolbar.tsx` — missing `data-testid="annotation-toolbar"`
- `apps/dentalemon/src/features/imaging/components/imaging-workspace.tsx` — missing `data-testid="imaging-toolbar"`, `data-testid="brightness-control"`, `data-testid="fullscreen-btn"`, `data-testid="save-measurement"`

**Impact:** E2E assertions for IMG-02, IMG-03, IMG-04, IMG-16 smoke tests will fail even if harness routes existed.

**Fix:** Add `data-testid` attributes to the wrapper elements in each component.
